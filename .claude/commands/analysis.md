# /analysis — 分析自動化 Skill

データを渡すと DoD に沿った分析を実装し、DA レビューが収束したら PR を作成する。

## 起動方法

```
/analysis <data_path_or_theme> [--title <slug>] [--rq "リサーチクエスチョン"] [--no-model]
```

**引数:**
- `<data_path_or_theme>` — データファイルのパス、URL、または分析テーマの文字列（必須）
- `--title <slug>` — ディレクトリ名に使うスラッグ（省略時は日付+テーマから自動生成）
- `--rq "..."` — リサーチクエスチョン（省略時は EDA 後に analysis-agent が提案）
- `--no-model` — モデリングフェーズをスキップする

**例:**
```
/analysis data/raw/population.csv --title population-aging --rq "少子高齢化の地域差はどれくらいか"
/analysis "東京都の気温と電力消費" --no-model
```

## 依存ファイル

- `.claude/rules/analysis-dod.md` — DoD（読み取り専用・唯一の真実）
- `.claude/agents/analysis-agent.md` — 分析実装エージェント
- `.claude/agents/da-reviewer.md` — DA レビュワーエージェント
- `.github/PULL_REQUEST_TEMPLATE.md` — 手動 PR 用テンプレート（自動 PR では body を動的生成）

## メインフロー

### Step 1 — 引数パース

`$ARGUMENTS` を解析して以下を決定する:

```
data_path  = <最初の位置引数>
title      = --title の値 OR "YYYYMMDD-" + data_path のファイル名（拡張子なし）
rq         = --rq の値 OR null
has_model  = --no-model が指定されていなければ true
```

今日の日付を YYYYMMDD 形式で取得し title の先頭に付与する。

### Step 2 — ブランチ作成

```bash
git checkout -b analysis/<title>
```

ブランチが既に存在する場合はそのままチェックアウトする（再実行の場合）。

### Step 3 — analysis-agent 起動（初回）

以下を入力として `analysis-agent`（`.claude/agents/analysis-agent.md`）を Agent ツールで呼び出す:

```
data_path      : <Step 1 で決定>
title          : <Step 1 で決定>
rq             : <Step 1 で決定>
has_model      : <Step 1 で決定>
review_feedback: null
```

`analysis-agent` から `ALL_PHASES_COMPLETE` を受け取るまで待機する。

### Step 4 — DA レビューループ

以下の変数で制御する:
```
review_count = 0
max_reviews  = 3
lgtm         = false
```

`lgtm == false` かつ `review_count < max_reviews` の間ループする:

1. `da-reviewer`（`.claude/agents/da-reviewer.md`）を Agent ツールで呼び出す
   - 入力: `analysis_dir = "analysis/<title>/"`, `review_count = review_count + 1`

2. 出力を受け取る:
   - `LGTM` を含む → `lgtm = true`、ループ終了
   - `NEEDS_REVISION` を含む → `feedback_text` に指摘内容を格納

3. `lgtm == false` の場合:
   - `review_count += 1`
   - `analysis-agent` を再呼び出しする
     - 入力: 前回と同じ + `review_feedback = feedback_text`
   - `REVISION_COMPLETE` を受け取るまで待機する

ループ終了後、`review_count >= max_reviews` かつ `lgtm == false` の場合:
- ユーザーに以下を報告して確認を求める:
  ```
  3 往復で LGTM が得られませんでした。
  現在の状態で PR を作成しますか？（手動レビューを推奨します）
  ```
- ユーザーが承認した場合のみ Step 5 へ進む

### Step 5 — PR body の動的生成

**ここで `.claude/rules/analysis-dod.md` を Read して PR body を組み立てる。**
DoD のチェック項目はこのファイルに書かない。以下の手順で動的に取得する:

1. `.claude/rules/analysis-dod.md` を Read する
2. セクション 1〜5 の `- [ ]` で始まる行をすべて抽出する
3. セクション見出し（`## N.` で始まる行）でグループ化する
4. `PROPOSAL.md` を Read して RQ セクションを抽出する
5. 以下の構造で PR body を組み立てる:

```markdown
## 分析概要

**リサーチクエスチョン:** {PROPOSAL.md から抽出}

**動機・背景:** {PROPOSAL.md から抽出}

---

## 成果物

{analysis/<title>/ 以下の実際のファイル一覧}

---

## DoD チェックリスト

> ソース: `.claude/rules/analysis-dod.md`（このリストはファイルから動的生成）

{analysis-dod.md Sec 1 見出し}
{Sec 1 の全 - [ ] 行}

{analysis-dod.md Sec 2 見出し}
{Sec 2 の全 - [ ] 行}

... （Sec 5 まで）

---

## DA レビュー履歴

- レビュー往復回数: {review_count}
- 最終判定: {LGTM / 未収束（手動レビュー推奨）}

---

## レビュワーへの補足

（特記事項があれば記載）

---

_Generated by `/analysis` skill. DoD: `.claude/rules/analysis-dod.md`_
```

### Step 6 — PR 作成

```bash
git add analysis/<title>/
git commit -m "analysis(<title>): add {RQ の先頭 50 文字}"
git push -u origin analysis/<title>
gh pr create \
  --title "analysis: {title} — {RQ の先頭 60 文字}" \
  --body "{Step 5 で生成した body}" \
  --base main
```

### Step 7 — 完了報告

ユーザーに以下を報告する:

```
分析が完了しました。

ブランチ: analysis/<title>
PR: {PR URL}

DA レビュー: {review_count} 往復で収束
成果物: analysis/<title>/REPORT.md
```

## エラーハンドリング

| 状況 | 対処 |
|------|------|
| `data_path` が存在しない | ユーザーにパスを確認して停止 |
| analysis-agent が `ALL_PHASES_COMPLETE` を返さない | エラー内容をユーザーに報告して停止 |
| `git push` が失敗する | エラーをユーザーに報告。force push は行わない |
| PR 作成が失敗する | `gh` のエラーをユーザーに報告。手動作成を案内する |
